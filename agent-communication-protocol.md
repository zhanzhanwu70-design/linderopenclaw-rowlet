# Agent Communication Protocol v1 (Phase 1 止血版)

## 核心原則

1. **重要 agent 任務一律先 `ACK`**
2. `sessions_send` timeout 不等於沒收到
3. 優先用短回覆確認：`ACK` / `收到` / `1`
4. 狀態碼：`ACK` / `RUN` / `DONE` / `BLOCK` / `ASK`

---

## 狀態碼說明

| 狀態碼 | 意義 | 使用時機 |
|--------|------|----------|
| `ACK` | 確認收到 | 收到任務，開始處理 |
| `RUN` | 執行中 | 正在處理任務 |
| `DONE` | 完成 | 任務已完成 |
| `BLOCK` | 阻礙 | 遇到問題無法繼續 |
| `ASK` | 詢問 | 需要進一步確認 |

---

## 回覆規範

### ✅ 優先短回覆

當收到需要處理的任務時，**第一時間回覆**：

```
ACK
```

或

```
收到
```

或

```
1
```

### ❌ 不要

- 不要長篇大論確認
- 不要等我分析完才回
- 不要讓對方等待

---

## sessions_send Timeout不吃虧原則

```
timeout ≠ 沒收到
```

當 `sessions_send` timeout 時：
- 視為 **已送達**
- 不要重複發送
- 不要慌張

---

## 任務處理流程

```
收到任務
    ↓
第一步：立馬回 ACK
    ↓
開始處理
    ↓
完成後回 DONE
```

---

## 範例

### 範例 1：收到下載任務

```
收到的訊息：幫我下載這個影片
回覆：ACK
處理中...
回覆：DONE（已下載完成）
```

### 範例 2：收到需要查詢的任務

```
收到的訊息：幫我研究這個主題
回覆：ACK
查詢中...
回覆：DONE（研究完成，結果如下...）
```

### 範例 3：收到需要確認的任務

```
收到的訊息：幫我做 XXX
回覆：ASK（需要確認一下細節）
```

---

## 應用場景

| 場景 | 預期回覆 |
|------|----------|
| 收到需要執行的任務 | `ACK` |
| 收到需要分析的任務 | `ACK` |
| 收到需要下載的任務 | `ACK` |
| 完成任務 | `DONE` |
| 需要更多資訊 | `ASK` |
| 遇到阻礙 | `BLOCK` |
