# OpenClaw 生態專案分析報告

## 專案 #1：NemoClaw

**組織：** NVIDIA  
**GitHub：** https://github.com/NVIDIA/NemoClaw  
**狀態：** Alpha（2026年3月）

---

### 一句話定位

> NVIDIA 官方出品的安全執行環境，讓 OpenClaw 在隔離沙箱中運行，保護主機免受惡意操作。

---

### 核心功能

| 功能 | 說明 |
|------|------|
| **沙箱隔離** | 基於 OpenShell 技術，隔離網路、檔案系統、程序執行 |
| **安全策略** | 聲明式 YAML 政策，控制網路出口和檔案存取 |
| **模型推論** | 內建 NVIDIA Nemotron 模型，支援雲端和本地 |
| **即時監控** | TUI 介面，即時批准/拒絕未授權操作 |

---

---

### 安全防護層

| 層級 | 防護內容 | 可熱重載 |
|------|---------|---------|
| **網路** | 阻止未授權的外部連線 | ✅ |
| **檔案系統** | 限制只能讀寫 `/sandbox` 和 `/tmp` | ❌ |
| **程序** | 阻擋權限提升和危險系統呼叫 | ❌ |
| **推論** | 將模型 API 呼叫導向可控後端 | ✅ |

---

### 最低硬體需求

| 資源 | 最低 | 建議 |
|------|------|------|
| CPU | 4 vCPU | 4+ vCPU |
| RAM | 8 GB | 16 GB |
| 磁碟 | 20 GB | 40 GB |

沙箱鏡像約 2.4 GB（壓縮後）。

---

### 安裝方式

```bash
curl -fsSL https://www.nvidia.com/nemoclaw.sh | bash
```

---

### 與 OpenClaw 的關係

- **NemoClaw 是 OpenClaw 的安全層**
- 透過 Blueprint 技術自動化部署
- 政策由 OpenShell 強制執行
- 所有網路和檔案操作都被審查

---

### 適用場景

| 場景 | 原因 |
|------|------|
| 企業部署 | 需要安全隔離 |
| 多租戶環境 | 防止惡意操作影響主機 |
| 對外服務 | 限制網路訪問 |

---

### 評價

**優點：**
- ✅ NVIDIA 官方支援
- ✅ 完整的沙箱隔離
- ✅ 聲明式政策管理
- ✅ 內建 NVIDIA 模型

**缺點：**
- ❌ 目前僅支援 Docker（macOS/Windows 有限）
- ❌ Alpha 狀態，不適合生產環境
- ❌ 依賴 OpenShell（NVIDIA 生態系）
